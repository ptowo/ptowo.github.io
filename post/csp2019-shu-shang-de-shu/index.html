CSP2019 树上的数

2022-06-02 树形结构 神仙题

题目描述

给定一个大小为 nnn 的树，它共有 nnn 个结点与 n−1n - 1n−1 条边，结点从 1∼n1 \sim n1∼n 编号。初始时每个结点上都有一个 1∼n1 \sim n1∼n 的数字，且每个 1∼n1 \sim n1∼n 的数字都只在恰好一个结点上出现。

接下来你需要进行恰好 n−1n - 1n−1 次删边操作，每次操作你需要选一条未被删去的边，此时这条边所连接的两个结点上的数字将会交换，然后这条边将被删去。

n−1n - 1n−1 次操作过后，所有的边都将被删去。此时，按数字从小到大的顺序，将数字 1∼n1 \sim n1∼n 所在的结点编号依次排列，就得到一个结点编号的排列 PiP_iPi​。现在请你求出，在最优操作方案下能得到的字典序最小的 PiP_iPi​。

如上图，蓝圈中的数字 1∼51 \sim 51∼5 一开始分别在结点②、①、③、⑤、④。按照 (1)(4)(3)(2) 的顺序删去所有边，树变为下图。按数字顺序得到的结点编号排列为①③④②⑤，该排列是所有可能的结果中字典序最小的。

Data Range:1≤n≤2000\text{Data Range:} 1 \leq n \leq 2000Data Range:1≤n≤2000。

暴力可以直接通过枚举全排列然后直接进行交换检验最后统计答案。

继续思考，我们其实一定会有一个贪心的思想，我们会先考虑数字 111 最终的节点，然后再考虑数字 222 的最终节点，依次往下考虑。

然后考虑菊花图的做法，假设度数为 n−1n-1n−1 的点为 uuu，n−1n-1n−1 条边为 (u,v1),(u,v2),(u,v3),…,(u,vn−1)(u,v_1),(u_,v_2),(u,v_3),\dots,(u,v_{n-1})(u,v1​),(u,​v2​),(u,v3​),…,(u,vn−1​)。

如果现在有一个排列 (p1,p2,…,pn−1)(p_1,p_2,\dots, p_{n-1})(p1​,p2​,…,pn−1​)，假设第 iii 次删边山的是 (u,pi)(u, p_i)(u,pi​)。

经过一个模拟，发现 ap1a_{p_1}ap1​​ 到了 p2p_2p2​，ap2a_{p_2}ap2​​ 到了 p3p_3p3​， apn−1a_{p_{n-1}}apn−1​​ 到了 uuu。

发现这样成了一个环的形式， rt→p1→p2→⋯→rtrt \rightarrow p_1 \rightarrow p_2 \rightarrow \dots \rightarrow rtrt→p1​→p2​→⋯→rt。

对于当前的数字 iii，我们从小到大的枚举这个数字，然后贪心的枚举自己环上的下一个位置，然后最后为了防止提前成环，使用一个并查集维护这个过程。

考虑链的做法，特点是每个点的最大度数都为 222。

对于一个数字 kkk，它要从 u1u_1u1​ 移到 umu_mum​，在路径上有一下性质：

  • 对于起点 u1u_1u1​，其出边 (u1,u2)(u_1,u_2)(u1​,u2​) 一定是这一点先被删掉的边。
  • 对于终点 umu_mum​，其入边 (um,um−1)(u_m,u_{m-1})(um​,um−1​) 一定这一点后被删掉的边。
    否则，当然已经将 kkk 换到了点 umu_mum​，但是所有边最后是都要删除的，因此如果出边 (um,um+1)(u_m,u_{m+1})(um​,um+1​) 后删的话，最后 kkk 会被换到点 umu_mum​ 上。
  • 对于中间点 uiu_iui​，其入边 (ui−1,ui)(u_{i-1},u_i)(ui−1​,ui​) 一定比 (ui,ui+1)(u_i,u_{i+1})(ui​,ui+1​) 先删。

因此每个点都能得到一个删边的顺序，左先于右或者右先于左。

定义 tagitag_itagi​ 表示每个点的删边情况，tagi=0tag_i=0tagi​=0 表示还没删，tagi=1tag_i=1tagi​=1 表示先删左边后删右边，tagi=2tag_i=2tagi​=2 表示先删右边后删左边。

一个直接的想法是，我们从小到大枚举要换的数字 valvalval，然后枚举这个数字的最终节点，然后检查这一段的 tagtagtag 是否与之前的会有冲突，如果没有冲突就换。

这样的复杂度为 n3n^3n3。

但是链的部分有 n≤2000n \leq 2000n≤2000，考虑如何优化到 n2n^2n2。

你发现这个扩展的方式其实存在一个单调性，比如往左边扩展的时候，如果有个点无法经过，那么更远的点就都无法经过。

于是我们直接搜出合法的点，然后取这里面最小的就好了，时间 n2n^2n2。

下面是树的解法，也是全题最 nb 的地方，将树形结构考到的极致，不得不说确实是神仙题目，看了官方题解被震撼到感觉 202020 年的饭白吃了，经过自己的理解，将简化思路放了出来。

注意到一件事情，和一条边 (u,v)(u,v)(u,v) 直接相关的只会是它相邻的边，也就是 uuu 的边和 vvv 的边。

然后如果不考虑边 (u,v)(u,v)(u,v) 的话，我们发现，对于点 uuu 其他的边 AAA 和点 vvv 其他的边 BBB，AAA 和 BBB 之间先后顺序关系没有影响。

那么我们可以先分别钦定 A,BA,BA,B 内部的删边顺序，然后钦定 (u,v)(u,v)(u,v) 在两个大的边集中的删边顺序。

于是我们可以将树上的问题转换成每个节点为一个单位的子问题。

对于在某个节点的边，有三种下列类型的条件：

1.1.1. 某条边被最先选择

2.2.2. 某条边紧更另一条边被选择

3.3.3. 某条边最后被选择

如果一个数字 kkk 要从点 uuu 的一条边 (u,v)(u,v)(u,v) 离开，那么这条边 (u,v)(u,v)(u,v) 一定是最先被选择的边。

如果一个数字 kkk 要从点 uuu 的 (u,v1)(u,v_1)(u,v1​) 进入，再从 (u,v2)(u,v_2)(u,v2​) 离开，那么 (u,v1)(u,v_1)(u,v1​) 一定比 (u,v2)(u,v_2)(u,v2​) 先被选择，并且 (u,v2)(u,v_2)(u,v2​) 紧跟 (u,v1)(u,v_1)(u,v1​)。

如果某个数字要最终停在一个点 uuu，那么要选择的边一定是最后被删除的边 (u,v)(u,v)(u,v)。

我们把一个点 xxx 的邻接边抽象成一堆点，然后对这个点构成的图重新加边，一条边 (u,v)(u,v)(u,v) 表示 (x,u)(x,u)(x,u) 比 (x,v)(x,v)(x,v) 先删除。

那么一个合法方案绝不会有一个代表一条边的点的出度或者入度超过 111。

这样我们用有向边表示删除的先后关系，最后形成的总是一堆链。

于是我们仍然从小到大枚举数 iii，像链一样用 dfs 扩展可行的点，记录可以走到的最小的点。

对于可走不可走有下列限制：

不能有边比当前的第一条边先删除，比当前最后一条边后删除

最后删除的先后关系形成的总是一堆链。

如果当前的操作会导致当前的第一条边和最后一条边合并在一条链里面，并且除开它们所在的链以外还有一些链没有合并起来，那么这就是不合法的（参考菊花图，这样会导致边删不完）

然后如此判断使用并查集和链的方法拼凑起来维护，即可做到 n2n^2n2，可以通过此题。

namespace subtask3 {
  int dfn[N], nfd[N], pt, vis[N], tag[N], ans[N];
  void dfs(int u,int f) {
    dfn[u] = ++pt; nfd[pt] = u;
    for (int i = first[u]; i; i = nex[i]) {
      int to = v[i]; 
      if (to == f)
        continue;
      dfs(to, u);
    }
  }
  //tag = 1 表示向左走，tag = 2 表示向右走
  int check_l(int p1,int p2) {
    if (tag[p1] == 1 || tag[p2] == 1)
      return false;
    for (int i = p1 + 1; i < p2; i++) 
      if (tag[i] == 2)
        return false;
    return true;
  }
  int check_r(int p1,int p2) {
    if (tag[p1 ] == 2 || tag[p2] == 2)
      return false;
    for (int i = p2 + 1; i < p1; i++) 
      if (tag[i] == 1)
        return false;
    return true;
  }
  void push_l(int p1,int p2) {
    if (p1 != 1 && p1 != n) 
      tag[p1] = 2;
    if (p2 != 1 && p2 != n)
      tag[p2] = 2;
    for (int i = p1 + 1; i < p2; i++)
      tag[i] = 1;
  }
  void push_r(int p1,int p2) {
    if (p1 != 1 && p1 != n)
      tag[p1] = 1;
    if (p2 != 1 && p2 != n)
      tag[p2] = 1;
    for (int i = p2 + 1; i < p1; i++)
      tag[i] = 2;
  }
  int find_l(int u) {
    int res = n + 1;
    if (tag[ dfn[u] ] == 2) 
      return res;
    for (int j = dfn[u] - 1; j >= 1; j--) {
      if (tag[j] == 1) {
        if (!vis[j])
          res = min(res, nfd[j]);
        break;
      }
      if (!tag[j] && !vis[j])
        res = min(res, nfd[j]);
    }
    return res;
  }
  int find_r(int u) {
    int res = n + 1;
    if (tag[dfn[u]] == 1) 
      return res;
    for (int j = dfn[u] + 1; j <= n; j++) {
      if (tag[j] == 2) {
        if (!vis[j]) 
          res = min(res, nfd[j]);
        break;
      }
      if (!tag[j] && !vis[j])
        res = min(res, nfd[j]);
    }
    return res;
  }
  void solve() {
    for (int i = 1; i <= n; i++)
      tag[i] = vis[i] = ans[i] = 0; pt = 0;
    for (int i = 1; i <= n; i++) 
      if (deg[i] == 1) {  dfs(i, 0); break; }
    for (int i = 1; i <= n; i++) {
      int rt = find_l(p[i]), tp = find_r(p[i]);
      if (rt < tp) 
        push_r(dfn[p[i]], dfn[rt]);
      else rt = tp, push_l(dfn[p[i]], dfn[rt]);
      ans[i] = rt; vis[ dfn[rt] ] = 1;
    }
    for (int i = 1; i <= n; i++)
      print(ans[i]), putc(' ');
    putc('\n');
  }

const int M = 3000;
namespace subtask4 {
  struct DSU {
    int fa[M], fir, lst, pre[M], nxt[M];
    void init(int n) {
      for (int i = 1; i <= n; i++)
        pre[i] = nxt[i] = 0, fa[i] = i;
      fir = lst = 0;
    }
    int getfa(int x) {
      if (fa[x] != x)
        fa[x] = getfa(fa[x]);
      return fa[x];
    }
    bool same(int x,int y) {
      return getfa(x) == getfa(y);
    }
    void merge(int x,int y) {
      x = getfa(x), y = getfa(y);
      fa[x] = y;
    }
  }t[M];
  int dfs(int u,int fa) {
    int res = n + 1;
    //deg[x] 表示剩下的 x 的没被删的边数
    //然后要么当前没有最后删除的边，要么最后删除的边已经是上一个
    if (fa && (!t[u].lst || t[u].lst == fa))
      if (!t[u].nxt[fa] && !(t[u].fir && deg[u] > 1 && t[u].same(fa, t[u].fir)))
        res = u;           //!(当前还有点没被删，但是此时首尾在一条链上)
      //最后删除的边没有后继
    for (int i = first[u]; i; i = nex[i]) {
      int to = v[i], id = (i >> 1); //当前边标号
      if (fa == id)
        continue;
      if (!fa) {
        if (!t[u].fir || t[u].fir == id) { 
          //起点判断，前一个转移过来的边是第一个被删除的边，或者还没有被删除的边
          if (t[u].pre[id]) //第一个删除的边有前驱就g
            continue;
          if (t[u].lst && deg[u] > 1 && t[u].same(t[u].lst, id)) 
          //有最后删边，并且还有没删的边，并且 last[u] 和起点边在一个集合
            continue;
          res = min(res, dfs(to, id)); //可以作为起点
        }  else  continue;
      }  else {
        if (t[u].fir == id || t[u].lst == fa || t[u].same(id, fa)) 
          continue;
        // 中间点判断
        if (t[u].pre[id] || t[u].nxt[fa])
          continue;
        // 当前的边有前驱了， 上一个点边后继了，那这条边不能更着他们删
        if (t[u].fir && t[u].lst && deg[u] > 2 && t[u].same(t[u].fir, fa) && t[u].same (t[u].lst, id))
          continue;
        //作为中间点判断
        res = min(res, dfs(to, id));
      }
    }
    return res;
  }
  bool push(int u,int f,int end) {
    if (u == end) {
      t[u].lst = f; return true;
    }
    for (int i = first[u]; i; i = nex[i]) {
      int to = v[i], id = (i >> 1);
      if (id == f)
        continue;
      if(push(to, id, end)) {
        if (!f)
          t[u].fir = id;
        else {
          t[u].nxt[f] = t[u].pre[id] = 1;
          deg[u]--; t[u].merge(f, id);
        } 
        return true;
      }
    }
    return false;
  }
  void solve() {
    for (int i = 1; i <= n; i++)
      t[i].init(n);
    for (int i = 1; i <= n; i++) {
      int res = dfs(p[i], 0);
      push(p[i], 0, res);
      print(res), putc(' ');
    }
    putc('\n');
  }
};
